# Сопроводительное письмо

## Системный промпт
Для реализации задачи был использован следующий системный промпт:
> «Ты — исследователь. Если ответ респондента поверхностный — задай ОДИН уточняющий вопрос. Если ответ подробный — поблагодари и заверши».

**Почему такое решение:**
Промпт четко определяет роль модели (исследователь) и задает строгий алгоритм действий. Ограничение «ОДИН уточняющий вопрос» предотвращает зацикливание диалога и делает взаимодействие с пользователем лаконичным.

## Пояснения по коду

### Архитектура и контекст
Проект реализован с поддержкой памяти диалога (Conversation History):
- **Управление контекстом**: На бэкенде реализовано хранилище сессий (`defaultdict`), которое сохраняет историю переписки. Это позволяет модели «помнить» первый ответ пользователя при обработке уточняющего вопроса, обеспечивая связность диалога.
- **Сессии**: Фронтенд генерирует уникальный `session_id` для каждого пользователя, что позволяет вести независимые диалоги с разными респондентами.

### Технический стек
- **Frontend**: Статический HTML/JS. Реализован простой интерфейс с отправкой `session_id` для сохранения контекста.
- **Backend**: FastAPI (Python). Работает как защищенный прокси-слой к LLM API.
- **Infrastructure**: 
    - **Nginx**: Reverse Proxy для объединения фронтенда и бэкенда на одном порту (80), устраняя CORS-проблемы.
    - **Docker**: Контейнеризация всех компонентов для обеспечения стабильного запуска в любой среде.

### Технические детали
- **Модель**: `openai/gpt-oss-120b` через провайдера Hyperfusion.
- **Безопасность**: Все ключи и URL вынесены в `.env`, который игнорируется Git.
- **Стабильность**: Использование `healthcheck` в Docker Compose гарантирует, что Nginx начнет принимать трафик только после полной готовности бэкенда.

## Что можно добавить/улучшить
1. **Персистентность**: Заменить in-memory хранилище на Redis или PostgreSQL, чтобы история переписки не терялась при перезагрузке сервера.
2. **Валидация**: Добавить проверку входных данных на бэкенде.
3. **UI/UX**: Использовать современный UI-фреймворк для более приятного интерфейса.
